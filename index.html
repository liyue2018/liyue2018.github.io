webpack 最基本的配置文件

1.webpack.config.js

package.json 文件

json 文件里不能写注释

当我们 在控制台，直接输入 webpack 命令执行的时候， webpack 做了以下几步：

1， webpack 发现 我们没有通过命令的形式，给他指定入口和出口
2， webpack 就会去项目的根目录中，查找 webpack.config.js 的配置文件
3， webpack 会解析执行这个配置文件， 当解析执行完配置文件后，就得到了配置文件中，导出的配置对象
4， 当webpack 拿到配置对象后，就拿到了配置对象中，指定的入口和出口 然后进行打包构建

2.使用 webpack-dev-server 来实现自动打包编译的功能

2.1 npm i webpack-dev-server -D 安装到项目的本地开发依赖
2.2 由于我们是在项目中安装的，所有无法把它当做脚本命令
2.3 webpack-dev-server 帮我们打包生成的bundle.js 文件，并没有存放到实际的物理磁盘上，而是直接托管到了电脑的内存的。
所有我们在项目的根目录中，根本找不到打包好的bundle.js

我们可以认为，webpack-dev-server 把打包好的文件，以一种虚拟的形式，托管到了项目的根目录下，跟dist src node_modules平级的，有一个看不见的文件，叫做 bundle.js

export default export

在 ES6中，通过规范的形式， 规定了ES6中如何导入和导出模块

ES6中导入模块，使用 import 模块名称 from ‘模块标识符’ （导入样式时） import ‘表示路径’

在ES6中，使用 export default 和 export 向外暴露成员，在一个模块中，可以同时使用两者。

export default 只允许向外暴露一次，可以使用任意的变量来接收

export 向外暴露的成员，只能使用 {} 的形式来接收，这种形式叫作【按需导出】,可以暴露多个成员，如果某些成员我们在import 时不需要，则可以不在{}z中定义。

注意：接收时要严格按照导出时的名称，来接收。

如果想用别的名称来接收，可以使用 as 来起别名 { title as title124}

在node中使用 var 名称 = require(‘模块标识符’)

module.exports 和 exports 来暴露成员

触屏事件

事件类型

touch 事件的触发，必须保证元素有具体的宽高值，否则不会触发

touchstart 手指触摸屏幕时触发
touchmove 手指在屏幕上移动时触发 持续触发
touchend 手指离开屏幕时触发

TouchEvent对象

touches：位于屏幕上的所有手指的列表
targetTouches：位于该元素上的所有手指的列表 (建议使用)
changedTouches：tuchstart 时包含刚与触摸屏的触点，tuchend 时包含离开触摸屏的触点

touches 和 targetTouches 是没有区别的

手指对象的坐标值：

screenX / screenY 是手指的触点相对于屏幕左上角的坐标距离
clientX / clientY 相对于当前视口—移动端屏幕 (建议使用)
pageX / pageY 相对于当前页面的内容–包含滚动条
